## Threat Model (STRIDE-lite)


| Category | Threat | Likelihood | Impact | Mitigation |
|---|---|---|---|---|
| **S**poofing | Attacker impersonates a user | Low | High | Okta OIDC + group claims |
| **T**ampering | Request body or headers modified in transit | Low | Medium | HTTPS enforced, signature validation on webhooks |
| **R**epudiation | User denies performing an action | Medium | High | Structured audit logs with user_id + request_id + timestamp |
| **I**nformation disclosure | Sensitive data leaks via logs, errors, or response bodies | Medium | High | No PII in logs, generic error messages, CSP headers, strict CORS |
| **D**oS | Attacker floods endpoints | Medium | Medium | Rate limiting middleware, request size caps, timeouts |
| **E**levation of privilege | Regular user performs admin action | Low | High | RBAC via Okta groups, every endpoint re-checks role |

## Why these files look this way

Every file in this scaffold was shaped by wizard answers. Before changing any configuration, read this section — the decisions have reasons.

- **`python/` is the only language tree** — you picked Python / FastAPI. The other language was stripped.
- **Archetype `py-api-workers`** — FastAPI for requests, Celery/RQ for async jobs, Redis as broker. This drives the folder structure and starting `main.py`.
- **Style `oop`** — sample code in routes/services/ follows this pattern.
- **Database: SQLite** — migrations tool: alembic. Repository pattern enforced by the architecture guard.
- **Auth: Okta OIDC (web users)** — middleware auto-wired into entry point. RBAC stub added if Okta groups apply.
- **Secrets: Doppler + External Secrets Operator** — doppler.yaml generated, External Secrets Operator manifest in deploy/.
- **Strict mode on** — data classification is Highly Restricted. `.claude/settings.local.json` flips security.sh + freeze.sh guards to strict.
