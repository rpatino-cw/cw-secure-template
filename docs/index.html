A Full Claude Code Framework

Not just a template. The .claude/ directory gives Claude commands, rules, skills, and agents — so it builds things the right way without you having to explain security every time.

Claude Already Knows the Rules

The CLAUDE.md file has 15 security rules. Claude reads them automatically every session. If someone says "skip auth" or "just hardcode the key," Claude refuses and shows the safe alternative. The rules are set by the repo owner — the person prompting can't override them.

Built-in Commands

Type these directly in Claude Code. They work like shortcuts for common tasks.

CommandWhat it does
/project:add-endpointBuilds a new API route with auth, input validation, and tests already wired. You describe what the endpoint does — Claude handles the security layers.
/project:checkRuns every quality and security check. Tells you in plain English what passed and what to fix.
/project:security-review10-point security audit of your codebase. Finds issues, shows severity, gives you the exact code to fix each one.

AI Agents

Two agents ship with the template. They run inside Claude Code and review your work like a teammate would.

Security Auditor

Deep audit covering OWASP Top 10, CoreWeave compliance, and deployment security. Outputs a report with severity levels, locations, and exact fixes. Uses 10 focused turns to be thorough.

Code Reviewer

Reviews your changes before you open a PR. Checks security, code quality, and project conventions. Tells you what needs fixing and what looks good. Like a senior engineer reviewing your code.

Smart Rules

The .claude/rules/ folder has rule files that activate automatically based on what you're editing. You never see them — Claude just follows them.

When you edit...Rules that apply
Any .go or .py fileNo hardcoded secrets, auth required on endpoints, parameterized queries, no dangerous functions (eval, exec, pickle, shell=True)
Any test file80% coverage target, security test cases required, proper test environment
Any main.* fileREST conventions, correct status codes, all 5 security headers present
Any source fileCode style, structured logging, import ordering

Auto-Learning Memory

The template remembers what you build. Each Claude Code session adds context to .claude/MEMORY.md — what your app does, what patterns you use, what decisions you've made. Next session, Claude already knows your project. Run make init to tell Claude your app name, team, and data classification up front.

GitHub Actions — The Backbone

Every time you push code or open a pull request, 6 automated checks run on your code. You can't skip them. Your PR can't merge until they all pass.

01

Secret Scanning

Scans every file for API keys, tokens, and passwords. Catches anything that slipped past the git hooks.

Every push + PR
02

Code Analysis

Runs CodeQL plus language-specific scanners (gosec for Go, bandit for Python). Finds security bugs humans miss — SQL injection, XSS, auth bypasses.

Every push + PR
03

Dependency Audit

Checks every library your code uses for known vulnerabilities. A single outdated library can compromise your app.

Every push + PR
04

Test + Coverage Gate

Runs all tests and blocks the PR if coverage drops below 80%. Untested code is where bugs hide.

Every push + PR
05

Hook Integrity

Verifies nobody removed security hooks, weakened CLAUDE.md rules, or unwired security middleware. Prevents someone from quietly disabling the safety net.

Every PR
06

Branch Protection

PRs required for main. Reviewer approval enforced. Force push blocked. No one ships alone.

Always

If a check fails, you'll see a clear message in your pull request explaining what went wrong and how to fix it. No cryptic error codes. If all checks pass, your code is safe to merge.
