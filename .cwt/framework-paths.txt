# framework-paths.txt — Paths managed by cw-secure-template.
#
# Single source of truth for the CWT upgrade flow. Paths are organized
# by strategy:
#
#   [replace] — copy wholesale from upstream. User should not edit these
#               locally; changes will be overwritten on upgrade.
#   [merge]   — three-way merge (base/theirs/ours). User can customize;
#               upgrade preserves local edits where possible.
#
# Format: one path per line. Directory paths (ending in /) will have
# their contents enumerated recursively at upgrade time. Blank lines
# and # comments are ignored.
#
# User-owned paths (python/, go/, frontend/, src/, tests/, docs/) are
# intentionally absent — never touched by upgrades.
